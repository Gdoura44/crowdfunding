@startuml
skinparam defaultFontSize 14
skinparam participantPadding 60
skinparam boxPadding 30

participant "BullMQ Trigger\n(risk-analysis queue)" as BullMQ
participant "n8n Workflow" as n8n
participant "Gemini API" as Gemini
participant "Backend Internal API\n(/internal/*)" as API
database "MongoDB" as DB

BullMQ -> n8n: Consume job (project data)
 n8n -> API: POST /internal/run-risk-analysis { projectId, title, description, category, fundingGoal, deadline }
 note right: Authorization: Bearer <secret>
 
 API -> Gemini: POST /v1beta/models/{flash-model}:generateContent (prompt)
 Gemini --> API: risk JSON (riskScore, riskLevel, successProbability)
 
 API -> DB: Update project\nstatus=UNDER_REVIEW\naiStatus=COMPLETED\naiAnalysis=...
 DB --> API: OK
 API --> n8n: 200 { ok:true, project:{...} }

alt failure (any node)
  API -> DB: Mark project.aiStatus = "FAILED"\n(project.status remains AWAITING_AI)
  API -> DB: Insert FailedWorkflowEvent (workflowType="risk-analysis")
  API --> n8n: 5xx/4xx with details
end

note right of BullMQ
  **BullMQ Trigger (Consumer):**
  - Listens to Redis queue "risk-analysis"
  - Jobs persisted even if n8n restarts
  - Built‑in retries (configurable in n8n)
end note

note right of n8n
   **Retry Policy (configured in n8n):** Exponential backoff: 1 min, 2 min, 4 min (max 3 retries).
   Gemini API returns 429 (rate limit) or 5xx errors – retry applies. After 3 failures,
   the backend records the failure in FailedWorkflowEvents.
end note
@enduml
