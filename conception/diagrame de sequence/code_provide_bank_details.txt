@startuml
actor "User (Creator)" as user
participant "Frontend" as FE
participant "Backend API" as API
database "MongoDB" as DB
participant "BullMQ Queue" as Queue

user -> FE: View payout(s), select "Provide Bank Details"
FE -> API: GET /api/payouts/{id}
API -> DB: Find payout by id where project.creatorId = user.id
' Only PENDING status (not AWAITING_BANK_DETAILS) =====
DB --> API: payout (status = PENDING)
API --> FE: payout details
FE --> user: Show form

user -> FE: Enter bank details, submit
FE -> API: PUT /api/payouts/{id}/bank-details { bankDetails }

API -> API: Validate bank details format
alt validation fails
  API --> FE: 400 "Invalid bank details: <reason>"
else validation passes
  API -> API: Encrypt bank details (AES-256-CBC)
  API -> DB: Start MongoDB transaction
  ' Set status to READY (not AWAITING_BANK_DETAILS) =====
  API -> DB: Update payout.bankDetails = encryptedValue, status = READY, bankDetailsProvidedAt = now()
  API -> DB: Insert AuditLog { actorId: user.id, actorRole: USER, action: "PROVIDE_BANK_DETAILS", targetType: "Payout", targetId: payout.id, details: {} }
  API -> DB: Insert Notification (in-app) for admin (type PAYOUT_BANK_DETAILS_REQUEST, relatedEntityId = payout.id)
  API -> DB: Commit transaction
  API -> Queue: Add job "send-email" with notificationId
  API --> FE: 200 OK
  FE --> user: "Bank details saved. Admin will review and release funds."
end

note right of API
  - Bank details are encrypted at rest using AES-256-CBC before storage.
  - Details are stored only for this payout, never in User profile.
  - After details are provided, payout status becomes READY.
  - Admin can now approve the payout.
  - A notification is sent to admin.
  - **Notifications:** In‑app notification stored in transaction; email queued after commit.
end note
@enduml
