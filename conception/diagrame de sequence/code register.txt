@startuml
actor User
participant "Frontend" as FE
participant "Backend API" as API
database "MongoDB" as DB
participant "BullMQ Queue" as Queue

User -> FE: Fill registration form
FE -> API: POST /api/users/register

API -> API: Validate input
API -> DB: Check if email exists
DB --> API: result

alt Email already exists
  API --> FE: 409 Conflict
  FE --> User: Show error
else Email available
  API -> API: Hash password (bcrypt)
  API -> API: Generate verification token (random string)
  API -> API: Hash token (bcrypt) → tokenHash
  API -> API: Set expiry (e.g., 24h)
  API -> DB: Insert user with { isActive: false, verifyTokenHash, verifyTokenExpiry }
  API -> Queue: Add job "send-email" with { userId, type: "verification", token }
  API --> FE: 201 Created
  FE --> User: "Check your email"

  User -> Email: Click verification link   ' "Email" here is the external email client, not a service
  Email -> API: GET /verify?token=...

  API -> DB: Find user by tokenHash (re-hash provided token)
  DB --> API: user (or null)

  alt Token invalid or expired
    API --> FE: 400 Invalid or expired link
  else Token valid
    API -> DB: Update user: isActive = true, remove token fields
    API --> FE: 200 Account verified
    FE --> User: Redirect to login
  end
end

note right of API
  **Notifications:** Email is sent via BullMQ queue.
end note
@enduml
