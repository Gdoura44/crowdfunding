@startuml
actor User as user
participant "Frontend (React)" as FE
participant "Backend API (Node.js)" as API
database "MongoDB" as DB

user -> FE: Open profile settings
FE -> API: GET /api/users/profile
API -> DB: Find user by id (from token)
DB --> API: user profile data (firstName, lastName, phone, riskPreference, preferredCategories)
API --> FE: profile
FE --> user: Show profile form

user -> FE: Modify fields
FE -> API: PUT /api/users/profile { firstName, lastName, phone, riskPreference, preferredCategories }  ' CHANGED: removed bankDetails

API -> API: Validate riskPreference enum
API -> DB: Update user.profile fields
API -> DB: Insert AuditLog { actorId: user.id, actorRole: USER, action: "UPDATE_PROFILE", targetType: "User", targetId: user.id, details: { changes } }
API --> FE: 200 OK
FE --> user: "Profile updated"

note right of API
  - Email and password cannot be changed here (separate flows)
  - Bank details are collected per payout when a project reaches its goal
end note
@enduml
