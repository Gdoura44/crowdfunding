@startuml
actor User as user
participant "Frontend" as FE
participant "Backend API" as API
database "MongoDB" as DB
participant "BullMQ Queue" as Queue

user -> FE: Clicks "Forgot password"
FE -> API: POST /api/users/forgot-password { email }
API -> DB: Find user by email
DB --> API: user

alt user exists
  API -> API: Generate reset token + hash it (store hash + expiry)
  API -> DB: Update user with reset token hash and expiry
  'Optional in‑app notification (requires adding PASSWORD_RESET_REQUESTED to enum)
  API -> DB: Insert Notification (in-app) for user (type PASSWORD_RESET_REQUESTED)
  API -> Queue: Add job "send-email" with { userId, type: "password-reset", token }
  API --> FE: 200 OK
  FE --> user: "Check your email"

  user -> FE: Clicks reset link
  FE -> API: GET /api/users/reset-password?token=...
  API -> DB: Verify token (re‑hash and compare)
  DB --> API: user (if token valid and not expired)

  alt token valid
    API --> FE: 200 OK (show reset form)
    user -> FE: Enter new password
    FE -> API: POST /api/users/reset-password { token, newPassword }

    ' ===== NEW: Password strength validation =====
    API -> API: Validate password strength (min 8 chars, at least one letter, one number)
    alt password too weak
      API --> FE: 400 "Password must be at least 8 characters and contain both letters and numbers"
    else
      API -> DB: Start transaction
      API -> DB: Update user password (bcrypt hash)
      API -> DB: Delete reset token (set tokenHash = null, expiry = null)
      API -> DB: Insert AuditLog { actorId: user.id, actorRole: USER, action: "RESET_PASSWORD", targetType: "User", targetId: user.id }
      API -> DB: Commit transaction
      API -> API: Invalidate existing sessions (optional: clear refreshTokens)
      API --> FE: 200 OK
      FE --> user: Redirect to login
    end
  else token invalid/expired
    API --> FE: 400 Bad request
    FE --> user: "Invalid or expired link"
  end

else user not found
  API --> FE: 200 OK (avoid email enumeration)
  FE --> user: "If email exists, a reset link was sent"
end

note right of API
  **Password strength:** Enforced during reset to prevent weak passwords.
  **Notifications:** Email is sent via BullMQ queue.
end note
@enduml
