@startuml
actor User as U
participant "Frontend" as FE
participant "Backend API" as API
database "MongoDB" as DB
participant "Gemini API" as Gemini

U -> FE: Ask question about project
FE -> API: POST /api/projects/{id}/chat { question }

API -> API: Rate limit check (10/hour per user/IP)
alt Rate limit exceeded
  API --> FE: 429 Too Many Requests
else
  API -> DB: Fetch project details (title, description, category, goal, currentFunding, deadline)
  DB --> API: project

  note right of API
    Build prompt:
    "You are a crowdfunding assistant. No financial advice.
    Project: {title} - {description}
    Goal: {goal}€, Current: {currentFunding}€
    Platform rules: cancellation within 5 min after payment,
    overfunding = full refund, expired projects = refund.
    User question: {question}
    Answer concisely."
 
    **Rate limit storage:**
  - Limits: 10 requests per hour per user (userId) and per IP address (fallback).
  - Storage: in-memory (express-rate-limit) — acceptable for PFE demo.
  - For production: prefer Redis store for shared limits across multiple backend instances.

  end note

  API -> Gemini: POST /v1beta/models/{flash-model}:generateContent (prompt)
  Gemini --> API: answer
  API --> FE: { answer }
  FE --> U: Display answer
end
