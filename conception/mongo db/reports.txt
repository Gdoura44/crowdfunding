// Indexes
db.reports.createIndex({ reporterId: 1 });
db.reports.createIndex({ projectId: 1 });
db.reports.createIndex({ status: 1, createdAt: -1 });

// Optional index for duplicate check optimization (if high report volume)
db.reports.createIndex({ reporterId: 1, projectId: 1, type: 1 });

// ============================================================================
// BUSINESS RULES (Enforced at Service Layer)
// ============================================================================
//
// **1. Self-Reporting Prevention:**
// reporterId MUST NOT equal project.creatorId
// A creator cannot submit a report on their own project.
// Validation performed before insert:
//   - Fetch Project by projectId
//   - Assert: reporterId !== project.creatorId
//   - If violated: throw "You cannot report your own project"
//
// **2. Duplicate Report Prevention:**
// A user can submit only ONE report per project per report type,
// unless their previous report was DISMISSED.
//
// Duplicate check query before insert:
//   db.reports.findOne({
//     reporterId: <userId>,
//     projectId: <projectId>,
//     type: <reportType>,
//     status: { $in: ["PENDING", "RESOLVED"] }
//   })
//
// If found: throw "You have already reported this project for {type}"
//
// **Why allow duplicates after DISMISSED?**
// - User's initial report may have been dismissed due to insufficient evidence
// - New evidence or repeated violations justify a new report
// - Only one ACTIVE report (PENDING/RESOLVED) per user per type is allowed
//
// **3. Resolution Requirement:**
// When admin changes status to RESOLVED or DISMISSED:
//   - resolution field MUST be populated (non-empty string)
//   - resolvedBy = admin's userId
//   - resolvedAt = current timestamp
//   - Validation Rule: When status is set to RESOLVED or DISMISSED, resolvedBy MUST NOT be null.
// If resolution is missing: throw "Resolution reason is required"
//
// **4. Post-Resolution Immutability:**
// Once status = RESOLVED or DISMISSED:
//   - Report cannot be edited (except by direct admin override)
//   - Prevents tampering with audit trail
//   - If new issues arise, user must submit a new report
//
// ============================================================================
